Analysis

If you want to insert a node at the beginning or at the end of the LinkedList, the complexity will be O(1) because at these positions, new references are easy to create to add nodes to the list.

To insert in an arbitrary place, going linearly is best option, so one would have to go through (n+1)/2 nodes. To insert the node, complexity is O(n) to reach the element and O(1) to insert it because the element just needs to be linked with the next and previous nodes.

Similarly, to delete a node, O(n) is the complexity to find the node and O(1) is the complexity to delete the node.

In ArrayLists, finding the point of insertion/deletion is O(1) and performing the actual insertion/deletion is O(n).

For the sort, the worst case scenario is O(n*n) because BubbleSort is used to sort the array of unsorted objects(cars). Since there are 30 objects that need to be sorted, the worst case time complexity is O(30*30).
