Overall Project Goal

Main Project Idea: Group Based Polling Application with Analytics

Theme and Concept:

  • Group based polling application with analytics
  • Allows admins to poll/quiz other users
  • Goal is have a unified area of data where users can create more than one form

Main User Interactions:

  • User can create an account (Spring Security)
  • Group admins can create a custom from for other users to fill out
  • Group admin can view form analytics through counters (Each entry represents a question and each row represents an answer choice)
  • Backend Database selection: MySQL driven by Hibernate POJO (SQL Repositories and JPA Repositories)

Key Technical Aspects:

  • Group admin creates a form which is saved and accessible by a certain key in the database
  • User completes a form which is loaded on a template page which consists of the question and selection responses which is actively pulled from the database when each page is loaded
  • As the user makes a choice, their response is stored corresponding to their user ID
  • Form content will be stored in two tables: 1) Table which stores the form ID, admin ID, and group ID (gives visibility only to given group) 2) Table that stores all of the questions which has rows for form ID (pulled from current form), question ID (generated), question itself, given response options, response ID
  • Form responses will be stored in a massive table in which each row contain the form ID, question ID, user ID, and the response the user selects (pulls response ID)

Features:

  • Login & account; save user in database using Spring Security
  • SQL management of user created data
  • Form responses: each column a question, each row a user response
  • Storing a new form, created by user
  • Each group: table for all of the form responses
  • Able to view the number of people that picked a specific answer for each question
